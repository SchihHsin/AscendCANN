1. 安装 CANN Toolkit

下载并安装适用于您硬件的 CANN 软件包。推荐使用 pip 安装 Python 绑定：

BASH
在 HiDevLab 运行
# 安装 CANN Python 包
pip install cann-toolkit
pip install torch-npu # PyTorch 适配
注意：需要提前在宿主机安装昇腾驱动（driver）和固件（firmware），版本需与 CANN Toolkit 匹配。

2. 第一个推理程序

以下示例演示如何使用 AscendCL 在昇腾 NPU 上运行 ResNet-50 推理：

PYTHON
在 HiDevLab 运行
# inference_demo.py
import acl
import numpy as np

# Step 1: 初始化运行时
ret = acl.init()
acl.rt.set_device(0)
ctx, _ = acl.rt.create_context(0)

# Step 2: 加载离线模型
model_id, _ = acl.mdl.load_from_file(
  "./resnet50.om"
)

# Step 3: 执行推理
input_data = np.random.randn(1, 3, 224, 224).astype(np.float16)
# ... 执行推理、获取输出 ...

# Step 4: 释放资源
acl.mdl.unload(model_id)
acl.rt.destroy_context(ctx)
acl.rt.reset_device(0)

ATC 模型转换工具

将训练框架导出的模型（ONNX、PB、Caffe 等）转换为昇腾离线模型（.om），并进行算子融合与图优化。

基本用法

BASH atc --model=resnet50.onnx \
  --framework=5 \ # 5=ONNX
  --output=resnet50 \
  --input_format=NCHW \
  --input_shape="input:1,3,224,224" \
  --soc_version=Ascend310

常用参数

参数说明默认值
--framework源框架类型：0=Caffe, 3=TF, 5=ONNX—
--input_shape输入张量形状，支持动态 shape：-1 表示动态维度—
--precision_mode精度模式：allow_fp32_to_fp16 / force_fp16allow_fp32_to_fp16
--op_select_implmode算子实现模式：high_performance / high_precisionhigh_performance

性能调优指南

使用 CANN Profiling 工具链深入分析 NPU 执行瓶颈，系统性优化推理与训练性能。

开启 Profiling

PYTHON from torch_npu.profiler import profile, ProfilerActivity

with profile(
  activities=[ProfilerActivity.NPU],
  record_shapes=True,
  profile_memory=True
) as prof:
  model(input_data)

print(prof.key_averages().table(sort_by="npu_time_total"))

API 参考

AscendCL API

完整的 C/C++ 与 Python 绑定接口文档，包含参数说明与使用示例。

INIT acl.init(config_path=None) 初始化 CANN 运行时

初始化 CANN 运行时环境。必须在所有其他 AscendCL API 之前调用。

参数类型说明
config_pathstr | None配置文件路径，传入 None 使用默认配置
PYTHON import acl
ret = acl.init() # ret == 0 表示成功
SET acl.rt.set_device(device_id) 指定当前线程使用的设备

将当前线程绑定到指定的 NPU 设备。

参数类型说明
device_idint设备 ID，从 0 开始编号
LOAD acl.mdl.load_from_file(model_path) 加载离线模型文件

从本地文件系统加载 .om 离线模型，返回模型 ID。

参数类型说明
model_pathstr.om 文件路径，支持绝对路径和相对路径

返回值：(model_id: int, ret: int) — 模型句柄和错误码

MEM acl.rt.malloc(size, policy) 在 NPU 侧分配内存

在昇腾设备的 HBM 内存上分配指定大小的缓冲区。

参数类型说明
sizeint分配字节数
policyint分配策略：ACL_MEM_MALLOC_HUGE_FIRST = 0
