摘要

这次评估没有做到“CANN 全量公开仓库完整盘点”。问题不在分析深度，而在当前公开可观测渠道本身不完整：

  • api.gitcode.com/api/v5/orgs/cann/repos 在当前环境需要 private-token，无法匿名列全量。
  • GitCode 组织页是前端渲染，静态 HTML 不直接给仓库列表。
  • 一部分仓库 git ls-remote 可访问，但 clone --depth 1 不稳定，出现只落 .git 或超时。
下面的判断基于“当前可观测并已验证的公开仓库样本集”，不声称覆盖了组织下所有仓库。

用户旅程与角色

Persona 1：样例消费型开发者

目标是尽快跑通一个 CANN 上的模型样例，不一定懂底层算子和集群通信。

代表仓库：cann-recipes-embodied-intelligence、cann-recipes-spatial-intelligence

Persona 2：模型 / 框架适配开发者

愿意做上游 repo 对接、模型改造、权重接入、推理调优。

代表仓库：cann-recipes-spatial-intelligence、cann-recipes-embodied-intelligence

Persona 3：算子 / 编译接口开发者

重点是写算子、调接口、理解 Ascend C / Python 编程模型。

代表仓库：pyasc、ops-math

Persona 4：系统 / 通信 / 基础设施开发者

重点是节点间传输、缓存分发、通信基础设施。

代表仓库：hixl、hcomm、asc-devkit

Agentic 评分结果

评分区间为 0-100。分数越高，说明 agent 越能在少人工干预的情况下自主推进。

  • 90-100：基本可自主推进
  • 70-89：可推进，但需少量人工补充
  • 50-69：能推进一部分，关键阶段会卡
  • 30-49：只能完成前半程
  • 0-29：几乎不可自主推进

分数怎么算

每个阶段分数都不是拍脑袋，而是按四项相加，满分 100：

组成项满分判断标准
文档清晰度30是否有明确、线性的动作链，是否写清版本、路径、成功判据。
环境可达性30当前常见开发环境能推进到哪一步，前置限制越少分越高。
自动化友好度25是否容易脚本化、agent 化推进，是否依赖手工复制、手找权重、跨仓拼装。
反馈与验证质量15是否有 help、日志、测试、benchmark、结果校验、失败反馈。

阶段分数 = 文档清晰度 + 环境可达性 + 自动化友好度 + 反馈与验证质量

角色总分怎么算

每个 Persona 的综合评分按旅程权重加权，不是简单平均：

阶段权重
搜索发现10%
代码获取10%
环境准备15%
依赖安装10%
数据/权重/外部资产准备10%
首次运行20%
结果验证10%
深入修改/扩展10%
排障与回归5%

角色总分 = Σ(阶段分数 × 阶段权重)

之所以把“首次运行”和“环境准备”权重放高，是因为这两个阶段最决定首次体验成败。

一个例子

以 Persona 1 的“环境准备 = 40 分”为例，这个分数大致对应：

  • 文档清晰度：18/30
  • 环境可达性：8/30
  • 自动化友好度：9/25
  • 反馈与验证质量：5/15

18 + 8 + 9 + 5 = 40

体验层面的核心问题

1. 入口碎

用户能看到很多 repo 名，但不知道哪个是总入口、哪个是骨架、哪个适合初学者。

2. 最大断点在“获取后第一小时”

问题不是 README 完全没有，而是 clone 不稳定、Linux/bash/conda/CANN/NPU 前置过重。

3. Recipe 更像文档集合

很多信息只存在自然语言里，没有结构化成 agent 可编排的元数据。

4. 获取稳定性拖累体验

多个 repo 能看到名字，但拿不到有效内容，这会直接破坏用户对整套体系的信任。

优化优先级建议

1. 统一入口索引页

按“入门 / 模型样例 / 算子开发 / 系统通信”四大入口组织，并标注仓库状态：可用、实验中、骨架、外部依赖重。

2. 增加机器可读 manifest

每个样例目录提供结构化元数据：平台、依赖、权重来源、最小命令、成功判据。

3. 提供无 NPU 预演路径

至少支持 doctor、check-env、download-assets、validate-layout。

4. 清理空壳 / 半可见仓库

未准备好的 repo 不应与成熟仓库出现在同一层入口。

5. 统一故障排查页

集中处理版本冲突、缺 bash/conda/npu-smi、权重缺失、上游 repo 未同步等高频问题。

如果只选一个仓库先做 agent 化体验样板，我建议先做 pyasc。如果要做业务开发者旅程主线，我建议先做 cann-recipes-embodied-intelligence。
