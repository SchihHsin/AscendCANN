①
三层增益框架 · 技术内核→开发界面→生态增益

→ 指向：三个楼层的分层标题徽章
②
三证据交叉矩阵 · 避免单视角盲区
洞察 · DX Core 4 · Spotify Soundcheck
体验健康矩阵把 Agent 评分 / 客户痛点 / VOD 声量三路信号并排。Agent 评分好但 VOD 声量高，说明用户实际感受更差；客户痛点少但 Agent 评分低，说明大客户还未充分暴露问题。
→ 指向：体验健康矩阵表格
③
角色视角筛选 · 聚焦而不隐藏
设计原则 · 渐进披露 · 减少认知负荷
资料负责人关注文档声量、算子编程组关注调试得分、社区运营关注入门旅程——同一份数据，不同角色的关注点差异显著。视角筛选淡化（而非删除）无关区域，保留完整上下文。
→ 指向：filter-bar 的视角 chips
④
需立即处理 · 把看板变成决策工具
洞察 · LinearB Scorecard · Datadog Watchdog
开发界面楼层单独提炼"需立即处理"清单，每条附优先级标签 + 责任分队 + 预期闭环时间。看板的终点是行动，不是观察——避免决策者在数十张图表中自行归纳行动项。
→ 指向：开发界面楼层左侧"需立即处理"卡片
