01 · 背景与目标
这个看板在解决什么问题？
明确核心用户、核心任务和核心问题，是重构的起点。

👤 看板的使用者

  • CANN 产品/体验团队负责人
  • 各角色方向的产品经理
  • 需要向上汇报体验数据的人

他们需要在高层次快速判断：哪里出了问题、问题有多严重。

🎯 核心任务

  • 快速定位体验最差的角色/场景
  • 了解每个痛点的来源（Agent测评 vs 真实用户）
  • 追踪改进后的数据变化

⚠️ 当前可能的痛点

  • 数据分散：评分/痛点/VOD 没有统一视图
  • 缺乏优先级感：不知道先改哪里
  • 角色切换体验割裂
  • 缺少"整体健康度"的直觉判断

💡 设计关键洞察

这个看板的核心价值不是"展示数据"，而是帮助决策者快速找到值得投入资源的改进点。

因此，视觉设计需要强调问题的严重程度（而非平均水平），并让 Agent评分、痛点、VOD 三类证据形成相互印证的关系。

🗂 两条并行的数据脉络

① 产品易用性（三类开发者）

关注各典型场景下的 Agent 评分和痛点密度，衡量"工具能不能用好"。

② 社区应用易用性（入门开发者）

关注端到端任务的完成率和障碍，衡量"新人能不能上手"。

04 · 交互设计方向
关键交互模式与设计原则
围绕"发现问题 → 定位原因 → 理解证据"这条主线设计交互。

🎛 筛选器设计

1

维度选择（横向 Tab）

产品易用性 / 社区应用易用性——决定进入哪条故事线

2

角色选择（卡片式选择器）

带角色图标和简短描述，让决策有上下文。被选中后高亮。

3

芯片型号（下拉/Tag选择）

多选支持跨型号对比，「全部型号」作为默认聚合视图

📈 数据展示设计

A

评分用色彩语义编码

红（<60）/ 黄（60-80）/ 绿（>80）——让严重问题一眼可见

B

场景卡片：密度感知

痛点数量用数字气泡显示，VOD 条数也要呈现，体现"证据厚度"

C

展开/收起的双层结构

场景列表是第一层，点击展开详情（Agent评分详情 + 痛点列表 + VOD原声）

D

VOD 作为"真实声音"单独呈现

引用样式，附来源标签，与 Agent 评分形成对照，增强可信度

🔢 全景层的优先级排序

所有场景按评分从低到高排列，最需要关注的永远在最上面。不要按场景名称字母序——那是数据库思维，不是决策思维。

📌 痛点聚类展示

同一场景下的痛点需要做归类（如：文档类、工具类、API类），避免长列表导致的信息过载。归类之后可以看出哪一类问题最突出。

📅 时间维度（建议后续加）

如果有历史数据，可以加"趋势线"：本期 vs 上期，直观呈现改进效果。这是看板从"报告工具"变成"追踪工具"的关键。

06 · 待确认问题
设计前需要和你确认的点
这些问题会直接影响最终设计方案，需要你的判断和补充。
Q1

Agent 评分的结构是什么？

评分是一个总分（0-100）？还是有多个子项（如：文档完整性/工具可用性/错误处理...）？每个场景的评分维度是否统一？这直接决定评分区域的展示方式。

Q2

痛点和 VOD 痛点的区别？

「痛点」是 Agent 跑出来的问题，还是人工整理的？「VOD 痛点」是真实用户反馈，那么 VOD 是什么来源（问卷/访谈/用户社区/工单）？两类痛点在看板上需要明显区分还是合并展示？

Q3

AI框架开发者和应用框架开发者的典型场景是什么？

目前只明确了算子开发者的 6 个场景。其他两种角色的场景结构是否类似？还是完全不同？这会影响页面是否能复用同一套模板。

Q4

芯片型号有哪些？是否需要跨型号对比？

目前提到了 A5，还有哪些型号？用户是否需要在同一个视图里对比「A5 vs A3」的评分差异？这会决定是否需要对比模式。

Q5

看板是否需要历史追踪功能？

现在是否有多期数据（如：2024-Q4 vs 2025-Q1）？如果有，这个看板是否需要展示趋势变化？这是纯静态展示还是接入实时数据？

Q6

入门开发者的任务是否有完成率数据？

5 个任务（搜索与发现、环境检查等）除了评分和痛点，是否有"Agent 是否完成该任务"的成功/失败状态？这会影响入门开发者视图的设计。

需要你提供的数据 / 素材

🔴 必需 核心数据

  • 各角色 × 场景的 Agent 评分（哪怕是示例数据或结构）
  • 痛点列表（文字描述 + 严重程度）
  • VOD 原声样本（哪怕 3-5 条）
  • 所有芯片型号名称
  • AI框架开发者 / 应用框架开发者的场景列表

🟡 可选 提升完整度

  • 品牌色 / 设计规范（如有）
  • 历史期数数据（用于趋势展示）
  • 角色图标或插图资源
  • 痛点分类体系（文档/工具/API等）
  • 看板预期的主要受众（团队内部/向上汇报/公开）
07 · 设计方向总结
建议的重构方向

核心设计原则

🎯
问题优先，而非数据优先 ——排序、颜色、强调都以"哪里最差"为导向，而非展示所有数据
🔗
三类证据互相印证 ——Agent评分 + Agent痛点 + VOD原声，在同一场景下并列呈现，增强可信度
📐
层级清晰，渐进披露 ——全景 → 角色 → 场景 → 详情，每一层只展示该层需要的信息
🧭
筛选即叙事 ——角色和芯片筛选不只是过滤数据，更是进入一个专属的分析视角
📈
为改进而设计 ——每个痛点和评分最终要指向"下一步做什么"，看板的终点是行动，不是观察
