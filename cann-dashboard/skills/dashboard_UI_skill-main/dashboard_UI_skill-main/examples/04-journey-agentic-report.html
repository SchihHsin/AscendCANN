Agentic 评分结果

评分区间为 0-100。分数越高，说明 agent 越能在少人工干预的情况下自主推进。

分数怎么算

阶段分数 = 文档清晰度 + 环境可达性 + 自动化友好度 + 反馈与验证质量

角色总分怎么算

角色总分 = Σ(阶段分数 × 阶段权重)

之所以把“首次运行”和“环境准备”权重放高，是因为这两个阶段最决定首次体验成败。

例子：Persona 1 的“环境准备 = 40 分”当前拆成：
  • 文档清晰度：9/30
  • 环境可达性：15/30
  • 自动化友好度：10/25
  • 反馈与验证质量：6/15

9 + 15 + 10 + 6 = 40

总体结论

CANN 当前的主要问题，不是“没有仓库内容”，而是“内容很多，但不同角色进入后，前 1 小时的体验链路不够顺滑”。

  • 最佳体验角色：算子 / 编译接口开发者，综合分 67.1
  • 最弱体验角色：样例消费型开发者，综合分 49.1
  • 共同断点：环境准备 / 首次运行
  • 这说明主问题不是“找不到内容”，而是“找到了也不容易顺利推进到成功”。
