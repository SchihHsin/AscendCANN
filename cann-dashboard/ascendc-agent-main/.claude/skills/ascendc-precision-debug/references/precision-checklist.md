# 精度检查清单

本文档提供贯穿开发全流程的精度检查要点，帮助在早期阶段预防精度问题。

---

## Plan 阶段

### 精度需求评估

- [ ] 已询问用户精度要求（默认 rtol=1e-3）
- [ ] 已识别精度敏感操作（累加、exp/log、归一化等）
- [ ] 已确定混合精度方案（哪些中间变量需要 FP32）
- [ ] 已明确验证标准和容差设置

### 精度敏感算子特征识别

| 特征 | 示例算子 | 建议方案 |
|-----|---------|---------|
| 累加类操作 | Reduce/Sum/Mean | 使用 FP32 累加器 |
| exp/log 操作 | Softmax/LogSoftmax | 设计数值稳定算法 |
| 归一化操作 | LayerNorm/BatchNorm | 中间值用 FP32 |
| 除法操作 | Div/Normalize | 添加除零保护 |
| 大数运算 | 矩阵乘法 | 避免溢出 |

---

## 实现阶段

### 精度意识编码规范

- [ ] 关键中间变量使用 FP32：`float sum_fp32 = 0.0f;`
- [ ] 数值敏感操作添加注释：`// 精度敏感：先减 max 避免溢出`
- [ ] 除零保护已添加：`abs(den) < eps ? 0 : num/den`
- [ ] 累加操作使用高精度累加器

### 常见精度陷阱预防

```cpp
// ❌ 错误：FP16 累加会损失精度
half sum = 0;
for (int i = 0; i < n; i++) {
    sum += input[i];  // 精度损失累积
}

// ✅ 正确：使用 FP32 累加器
float sum_fp32 = 0.0f;
for (int i = 0; i < n; i++) {
    sum_fp32 += static_cast<float>(input[i]);  // 保持精度
}
```

## 验证阶段

### 精度测试顺序（按优先级）

| 优先级 | 测试类型 | 目的 | 容差参考 |
|-------|---------|------|---------|
| 1 | 32字节对齐 + FP32 | 排除对齐和精度问题，验证算法 | rtol=1e-5, atol=1e-6 |
| 2 | 32字节对齐 + FP16 | 验证 FP16 精度 | rtol=1e-3, atol=1e-4 |
| 3 | 非对齐场景 | 测试非对齐处理 | 同上 |
| 4 | 边界值 | 边界条件（0、极大值、极小值） | 同上 |

### 对齐计算参考

| 数据类型 | 每元素字节数 | 32 字节对齐元素数 |
|---------|-------------|-----------------|
| FP16 | 2 字节 | 16 个元素 |
| FP32 | 4 字节 | 8 个元素 |
| INT8 | 1 字节 | 32 个元素 |

### 验证检查项

- [ ] 按 FP32 对齐 → FP16 → 非对齐顺序测试
- [ ] 边界值已测试（0、Inf、-Inf、极大/极小值）
- [ ] 精度在容差范围内
- [ ] 误差分布分析完成（识别系统性偏差）

---

## 调试阶段

### 调试前置检查

- [ ] 已固定最小可复现用例（shape/dtype/tensor值）
- [ ] 已保存固定输入数据到 test_data/
- [ ] 已生成 golden 输出用于对比

### 调试方法选择

```
调试计数器 = 0
    │
    ├─ 尝试快速方法（计数器 +1）
    │   ├─ 误差分布分析
    │   ├─ Printf 特定位置
    │   └─ 常见陷阱排查
    │
    └─ 计数器 >= 7 或方法穷尽 → 切换二分调试
```
